## 项目定位

这是一个静态小红书电商选品分析看板。页面逻辑和样式集中在 `index.html`，展示数据集中在 `xhs_product_data.js`。

## 目录与文件约定

- `index.html`：看板页面、交互逻辑、样式与图表渲染。更新数据时默认不改此文件。
- `xhs_product_data.js`：唯一数据入口。新增、替换、修正看板数据时优先只改此文件。
- `README.md`：项目说明。只有当运行方式或数据维护方式变化时才更新。
- `scripts/`：数据生成脚本。用于把原始 CSV 转成 `xhs_product_data.js`，不要在页面里手工复算复杂指标。
- 不新增临时导出、截图、原始表格到项目根目录；需要中间文件时放到项目外的工作目录，确认无用后清理。

## 数据更新规则

- 更新数据前先确认数据来源、统计日期和字段口径。
- 保持 `DATA` 对象结构稳定，避免改动前端读取字段名。
- 金额、销量、百分比等展示字段沿用现有格式，例如 `¥87`、`33,458`、`26.8%`。
- `meta.date` 和 `meta.generated` 必须与本次数据生成日期一致。
- 原始 CSV 不放入项目根目录；使用 `scripts/build-data.js` 读取外部 CSV 并生成数据文件。

## 验证方式

- 修改后优先用本地静态服务打开页面验证，不直接假设 HTML 可用。
- 重点检查：
  - 页面可以正常加载 `xhs_product_data.js`
  - 控制台没有 JavaScript 报错
  - 顶部统计、榜单、图表数据与新数据一致
  - 桌面和移动端布局没有明显遮挡或溢出

## 操作边界

- 不提交密钥、账号、Cookie、导出的私密原始数据。
- 不做 `git push`、发布、部署、删除文件或重写 git 历史，除非所有者明确要求。
